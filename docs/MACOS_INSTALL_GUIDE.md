# macOS 安装指南

## 🚫 "应用已损坏"问题解决方案

如果在 macOS 上遇到 `"无念剪映小助手"已损坏，无法打开。你应该将它移到废纸篓。` 的错误，这是 macOS 安全机制导致的，不是应用本身的问题。

## ⚡ 快速解决（推荐）

**对于 Apple Silicon Mac (M1/M2/M3/M4)，最可靠的方法：**

1. 打开 **终端** (Applications → 实用工具 → 终端)
2. 复制粘贴以下命令并按回车：
    ```bash
    sudo xattr -rd com.apple.quarantine /Applications/无念剪映小助手.app
    ```
3. 输入管理员密码
4. 完成后即可正常运行应用

## 🔧 解决方法

### 方法一：系统设置（推荐）

**macOS Ventura 13.0+ / Sonoma 14.0+:**

⚠️ **重要**: "仍要打开"选项只在特定条件下出现！

1. **先尝试运行应用** - 双击应用图标，让系统显示安全警告
2. **在警告出现后 1 分钟内** 打开 **系统设置** → **隐私与安全性**
3. 向下滚动找到 **安全性** 部分
4. 此时应该能看到被阻止的应用和 **仍要打开** 按钮
5. 如果没有看到，说明时间过了，需要重新触发或使用其他方法

**较旧的 macOS 版本:**

1. **打开系统偏好设置** → **安全性与隐私**
2. 在 **通用** 标签页中，找到被阻止的应用信息
3. 点击 **仍要打开** 按钮

### 方法二：右键菜单

1. **右键点击** 应用图标
2. 选择 **打开**
3. 在弹出的警告对话框中点击 **打开**

### 方法三：终端命令（最可靠）

```bash
# 方式1: 针对已安装的应用
sudo xattr -rd com.apple.quarantine /Applications/无念剪映小助手.app

# 方式2: 针对 DMG 文件（安装前处理）
sudo xattr -rd com.apple.quarantine ~/Downloads/WnksJianYingClient-v*.dmg

# 方式3: 如果应用名称包含空格，使用引号
sudo xattr -rd com.apple.quarantine "/Applications/无念剪映小助手.app"
```

**使用步骤:**

1. 打开 **终端** 应用 (Applications → 实用工具 → 终端)
2. 复制粘贴上述命令之一
3. 按回车键，输入管理员密码
4. 完成后即可正常运行应用

## 🛡️ 安全说明

- 这个错误是因为应用未经过 Apple 官方签名认证
- 应用本身是安全的，只是缺少付费的开发者证书签名
- 使用上述方法后，应用可以正常运行

## 📋 完整安装步骤

1. **下载** DMG 文件 (`WnksJianYingClient-v*.dmg`)
2. **双击** 打开 DMG 文件
3. **拖拽** 应用到 Applications 文件夹
4. **首次运行** 时会弹出安全警告
5. **使用上述方法之一** 解除安全限制
6. **后续使用** 可以正常启动

## 💡 为什么会出现这个问题？

- macOS Gatekeeper 默认只允许运行已签名的应用
- Apple Silicon Mac (M1/M2/M3/M4) 的安全检查更严格
- 个人开发者签名需要每年支付 $99 USD
- 未签名应用需要用户手动确认安全性
